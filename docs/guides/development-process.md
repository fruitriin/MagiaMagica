# 開発プロセス

```
計画（Plan）→ 実装 → 品質検証 → コミット
```

## 基本原則

- **計画駆動**: コードではなく計画をレビューする。筋の良い計画は受け入れ、実装は AI が担保する
- **ブートシーケンス**: セッション開始時に Feedback → TODO → Progress を順に読み込み、現在の状態を把握
- **品質ゲート**: ビルド・Lint・テスト → コードレビュー → セキュリティレビュー（オプション）
- **並列実装**: git worktree を活用したサブタスクの並列実行

## ブートシーケンス

セッション開始時に自動実行される手順:

1. `.claude/Feedback.md` を読む — 未対応の改善アクションを確認
2. `TODO.md` を読む — タスクバックログと優先度を把握
3. `.claude/Progress.md` を読む — 進行中タスクがあれば継続
4. TODO に未完了タスクがない場合 — オーナーに次のタスクを確認
5. knowhow サブエージェントを起動 — Plan に関連するノウハウをフィルタリング

## タスクのライフサイクル

### 開始時
- Feedback.md の改善アクションを確認
- サブタスクをチェックリストとして `.claude/Progress.md` に記録
- 並行作業できる粒度に分割

### 実装中
- サブタスク完了時にチェック
- 並列可能なタスクは git worktree で分離

### 完了時（品質検証）
1. ビルド・Lint・テスト実行
2. コードレビュー（`addf-code-review-agent`）
3. コントリビューション検出（`addf-contribution-agent`）
4. レビュー指摘への対応（Critical/High は必須修正）
5. Plan に実装結果を反映
6. Feedback.md に改善アクションを記録
7. Progress.md をアーカイブ
8. コミット

## 自律開発ループ

```
/loop 1h /addf-dev
```

AI エージェントが自動でタスクを選択し、上記のライフサイクルを繰り返します。
タスク選択の優先順位: ブロッカー解消 > インフラ整備 > 若番。

詳細は [CONTRIBUTING.md](../../CONTRIBUTING.md) を参照してください。
