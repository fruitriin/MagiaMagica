## 目的

過密な関数 (補助リング5個以上・glyph 10個以上) でも図形が重ならないようレイアウトを改良する。
**最重要制約: write_document 級の現行の見た目を変えない**(オーナー合格済みの意匠を守る)。

## スコープ

### やること

1. **兄弟 AuxRing の角度配分の動的化**
   - 現状: 同一 anchor の兄弟は固定ステップ 0.35 rad の扇状 → 4アーム以上で重なる
   - 改良: 兄弟数とリング半径から必要角度幅 (`2 * asin((r_child + margin) / distance)`) を計算して配分
2. **収まらない子の距離伸長**
   - 親リング1周に収まらない数の子は、距離を段階的に伸ばす (第2軌道)。`SigilId` 順で決定論的に
3. **glyph ファンの空き弧への退避**
   - 現状: 全周等間隔 → 隣接リングの上に落ちる
   - 改良: 兄弟 AuxRing が占有する角度帯を避けた空き弧に等間隔配置
4. **シグネチャ円弧の衝突回避**
   - 上半円に AuxRing がある場合、円弧半径を最外要素の外側まで広げる (固定オフセット → 動的)
5. **重なりメトリクスのテスト化**
   - 円どうしの交差数を数えるヘルパーを layout テストに追加し、過密 fixture で「重なり 0 (または明示した許容数)」を受け入れ基準にする
6. **fixture 拡充**
   - `fixtures/` に過密サンプル (`dense_dispatch.rs`: match 6アーム + 入れ子 + 呼び出し多数) を追加
   - 中規模の見た目回帰確認: write_document 相当の合成 fixture を追加し、変更前後の SVG を qlmanage で目視比較 (大きく変わらないこと)

### やらないこと

- force-directed レイアウト (Phase 2 以降の検討事項のまま)
- ノード重なりの完全な一般解 (Phase 1 系は同心円構造のヒューリスティックで足りる範囲まで)
- リング直径への cardinality.weight 反映の本格化 (Phase 2 のレイヤー作業と合流)

- 決定論性 (spec §6.1.4) は全改良で維持する。乱数・反復収束は使わず、閉形式の角度計算 + SigilId タイブレーク
- 定数 (`layout/constants.rs`) の意味は変えず、「動的計算の下限/上限」として再利用する
- ゴールデンスナップショット (magia-rust) は更新されるが、**write_document 級の中規模 fixture で
  位置の変化量をアサートする回帰テスト** (中心座標の移動が閾値以下) を入れて意匠の保全を機械化する

## 実装結果メモ (2026-06-11)

### 計画からの追加実装 (実装中に必要と判明)

- **入れ子の半円展開制限**: 計画の4項目だけでは入れ子リングが祖父母方向へ戻って衝突した。
  親が AuxRing の場合は outward 中心の半円にビン中央写像 `(anchor+0.5)/len` で展開
  (`anchor/len` だと単独子が真横に出る)
- **従兄弟リングの緩和パス** (`relax_ring_overlaps`): 角度配分は同一親内しか解決できない。
  重なったペアの大きい SigilId 側を過押し付きで押し出す固定32パス。重なりゼロなら no-op
- **glyph 回避帯の対象を全リングに拡大**: 自分の子だけでなく祖父母・叔父リングも
  半径方向フィルタ (|R-g| ≤ 半径和+margin、幾何学的に厳密) で対象化

### 設計判断の確定

- glyph は「全面再配分」ではなく**衝突個体のみ帯出口へ退避**。無衝突ケースは旧実装と
  bit 一致 → 中規模ベースライン回帰テストがそのまま意匠保全の機械化になった
- 過密時フォールバックは「候補へ全戻し」でなく「空き弧への均等配置」(段階的劣化)。
  全戻しは1個の超過で回避済みの衝突まで復活させる
- シグネチャ円弧はラベル帯 (±10px) に半径方向で食い込む上半分要素がある場合のみ反復拡張

### レビュー対応 (Stage 2)

- 修正: 緩和パスに過押し 0.5px + パス上限 32 (接触フリッカ防止、H-1)、
  min_angular_step の定義域防御 (M-3)、resolve_angles の前提条件 debug_assert (M-1)
- 誤検出と判断し根拠をコメント化: H-2/L-4 (半径方向フィルタは円対の接触可能性の厳密判定。
  祖父母リングの包含は glyph-main 衝突防止のための意図)、M-2 (TAU 超過後の詰め配置は意図)、
  M-4 (フォールバックへの rotation 反映は交差最小化のための意図、純関数なので不安定性なし)
- 既知の限界 (ドキュメント化済み): 病的密集での緩和パス残存重なり / glyph どうしの
  親またぎ重なり (スコープ外)。write_control_flow 級は目視で重なり解消を確認済み
