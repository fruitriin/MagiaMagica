# Phase 4.7 — 魔力回路アニメーション (実行順を流れる点P)

## 出典

- オーナー要望 (2026-06-12): 「処理のエントリポイントからステップインで制御フロー上を
  点Pが動く。非同期処理は点が増えるともっと面白そう」

> 本計画は **イメージ感の段階**。着手時に精緻化する。

## イメージ

関数の入口から「魔力の粒 (点P)」が実行順に魔法陣を流れる:

- メインリングの入口 (12時 — 4.0.6 後半の入口サインと同じ位置) から流れ始め、
  操作ドットを順に灯しながら進む
- 制御構造のドットに着くと**接続線を伝って補助陣へ降り**、腕の中を巡って戻ってくる
  (ステップインの体験)
- **分岐 (if/match)**: 点が腕の数だけ**分裂**して各腕を流れ、合流点で1つに戻る
  (静的近似 — どの腕を通るかは実行しないと分からないので「全部の可能性が流れる」
  という魔法陣らしい見せ方に倒す)
- **ループ**: 本体を 2〜3 周して出る (周回の演出)
- **非同期**: `await` 点・spawn 系の召喚印で**点が増える** — 並行して流れる粒が
  合流 (`join`/`await`) で戻る。async リング (内側二重線) では粒が常に複数泳ぐ、等
- 召喚印 (外部呼び出し) では粒が印に触れて一瞬光る (効果カテゴリ色のフラッシュ)

## 技術の土台 (全て完成済み)

- **実行順**: ベルカ式の anchor 順走査 (`belka.rs::visit_ring` — リングの Operation を
  順に辿り、制御構造の直後にその AuxRing 本体へ降りる) がそのまま点Pの経路定義
- **幾何**: 操作ドット座標・エッジ端点・リング半径は配置済み IR にある —
  Rust が「ウェイポイント列 (経路 IR)」を出し、Vue が動かす (POSD 分担そのまま)
- **非同期情報**: `ConcurrencyInfo` (is_async / await_points) は IR に積載済み
- **描画**: Vue 1本化 (4.3) 済みなので実装は Vue コンポーネントだけ。
  SVG `<animateMotion>` (SMIL) か rAF — 着手時に選定 (reduced-motion 対応必須)

## スコープ感 (着手時に精緻化)

- 経路 IR: `circuit: Waypoint[]` (座標 + 種別: dot 通過 / 線移動 / 分裂 / 合流 / 周回)
- Vue: 点Pの描画 + 再生コントロール (再生/停止/速度。「ステップ実行」 — 1操作ずつ
  進めるボタン — はオーナーの「ステップイン」の言葉から有力候補)
- 静止画 (SSR) では最終状態のみ (アニメは serve の動的 UI 専用)
