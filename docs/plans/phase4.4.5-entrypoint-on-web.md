# Phase 4.4.5 — エントリポイント (監視ファイル) を Web 上から変更する

## 出典

- オーナー要望 (2026-06-12、4.3.7 試遊時): 「エントリポイントも Web 上から変更できると
  うれしい」 — 現状は `magia serve <FILE>` の起動引数 (または .claude/launch.json) を
  書き換えて再起動するしかない
- **Phase 4.5 (ワークスペース俯瞰) からの先行切り出し**: 4.5 は「複数ファイルの俯瞰図」
  という大きな絵だが、「serve の監視対象を切り替える」だけならサーバの watch 切替 +
  ファイル選択 UI で完結する — 俯瞰ビューを待つ必要がない

> 本計画は **イメージ感の段階**。着手時に精緻化する。

## イメージ

- ヘッダのファイル名 (今は表示だけ) をクリック → ワークスペース内の `.rs` 一覧から選択
- 選択すると serve が監視対象を切り替え (notify の watch 張り替え + スナップショット
  再構築)、SSE で全クライアントが追従
- URL に `?file=<repo相対パス>` を載せ、リロード・共有で再現 (クエリ軸の規約どおり)

## スコープ感 (着手時に精緻化)

- serve: `POST /file` or `?file=` — 監視対象の切替。対象は**起動ディレクトリ配下の
  .rs に制限** (任意パス読み出しにしない — セキュリティ境界を明確に)
- ファイル列挙: 起動 dir 配下の `.rs` を glob (target/ 除外)。将来は cargo metadata
- Vue: ヘッダのファイル名をドロップダウン化 + URL 同期
- `?pin=` との整合: ファイル切替時は先頭関数へ fallback (既存の挙動を流用)
- diff (`?diff=`) はファイル切替後もそのまま効く (gitio はファイルパスから解決)

## 依存・位置づけ

- 単独で実施可能 (4.4 とも独立)。4.5 (俯瞰) はこの「複数ファイルを知る serve」を
  土台にできる
- FunctionToc の「ツリー表示 (複数ファイル対応後)」無効チェックボックス (4.1) は
  本計画 or 4.5 で有効化する

## 実装結果メモ (2026-06-12 完了)

- 切替はサーバ状態 (`POST /file`) として実装 — watcher リソースが絡むため
  ?pin 系のクライアント状態と区別した。URL `?file=` は「希望値」で、POST 応答の
  正規化パスで上書きしてループを断つ
- watcher は FS イベントと切替を **1 channel (WatchMsg enum)** に統合
  (std mpsc に select が無い)。レビューが2件の競合バグを検出:
  - W-1: デバウンスループが Switch を黙って捨てる → 退避して reload 後に処理
  - W-2: ファイル名と関数一覧が別ロックでずれる → **file を GoodSnapshot に
    統合して1ロックの一括スワップに** (diff の gitio 解決も同一スナップショットから)
- e2e fixture は cwd 相対起動に変更 (/files の境界 = cwd)。実リポジトリでは
  69 ファイルが候補に出る
- FunctionToc の「ツリー表示」有効化は 4.5 に残す (一覧 API はできた)
