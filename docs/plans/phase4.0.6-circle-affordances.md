# Phase 4.0.6 — 魔法陣の読み方アフォーダンス (凡例・入口サイン・補助陣ラベル)

## 出典

- オーナー要望 (2026-06-11、Phase 4.0.5 M3 判定時): ペアビューでコードと魔法陣を並べて見た際に出た「読み方の補助」要望3点。いずれも「別のスコープで OK」指定のため独立計画として記録
- 親計画: Phase 4.0.5 (ペアビュー) の上に乗る視覚改善。Phase 4.0.7 / 4.0.9 (Vue スキーマ化) と実装層が交差する

## 目的

魔法陣を初見・コード並置で読むときの「視線の迷子」を解消する。

1. **シンボル凡例リスト**: 色の意味 (効果カテゴリ 6色 / diff / ベルカ三極)、記号の意味 (`●` 操作ドット、`▲` 分岐、リング種別など) を UI 上で参照できる凡例パネル
2. **関数の入口と回転方向のサイン**: 実行順がどこから始まりどちらに回るかを図上に示す (現状は図形だけ見ると視線の移動方向が迷子になる)
3. **補助陣 (セカンダリ魔法陣) の関数名ラベル**: 呼び出し先関数の名前を補助リング上に表示する (コードと並べたときに対応が取れるように)

## スコープ

### やること

- **凡例パネル (`<SymbolLegend>`)**: Vue コンポーネント。`uno.config.ts` の theme (= palette.rs と同語彙) から色を引き、記号は小さな SVG サンプルで描く。開閉式 (常時表示は画面を食う)。**レンダラ非依存なので 4.0.7/4.0.9 を待たず実装可能**
- **入口・回転方向サイン**: メインリングの開始操作の位置に入口マーク、実行順の回転方向 (時計回り/反時計回り) を示す矢印 or グラデーション。spec の視覚語彙 (§6.1) への追記を含む
- **補助陣の関数名ラベル**: 補助リングの円弧に沿わせた小さい関数名 textPath (シグネチャ円弧の縮小版)。衝突回避 (Phase 1.8) との干渉に注意

### 実装層の判断 (オーナー確定 2026-06-11)

- **SVG 描画系 (入口サイン・補助陣ラベル) は 4.x 後半の Vue SSR 一本化 (Phase 4.3) の後に実施する**。Rust レンダラには足さない (4.3 で削除予定のコードを太らせず、Vue ビルダでの二度書きを避ける)
- **凡例パネルはレンダラ非依存のため先行可** — 4.0.5 完了後の任意のタイミングで着手できる
- 本計画は実質2段階: 前半 (凡例) / 後半 (サイン・ラベル、4.3 後)

## 実装結果メモ — 前半 (凡例パネル、2026-06-11 完了)

- `web/src/components/SymbolLegend.vue`: 開閉式 (既定閉)、**魔法陣ペインの下に配置** (オーナー判定 — 図と見比べながら読める位置。横長ペインに合わせ節を flex-wrap で横並び3カラム化)
- 内容: 効果カテゴリ6色 (書き起こしと同語源) / 図形8種 / 操作の強調 (ホバー=シアン・選択=金) / ベルカ三極。async 二重線は単体サンプル不可のため省略 (コメント記載)
- **記号サンプルは実描画コンポーネント (circle/*) を小 viewBox で再利用** — 意匠変更に凡例が自動追従。`selectable: false` でクリック・ホバーとも無効 (レビュー指摘で OperationDot/GlyphDot の mouseenter を selectable ガード — 凡例ホバーが focus store を汚染しない)
- Playwright 1本 (既定閉 → 開で各節が見える)
- レビュー: Critical/High 0、Medium 2 (branch サンプルの viewBox 1px / store 汚染) 修正済み、Low 3 対応済み

## 実装結果メモ — 後半 (入口サイン + 補助陣ラベル、2026-06-13)

- **入口サイン**: メインリングの 3 時に内向き三角 + 反時計回りのごく短い矢。
  実行順の出発点と向きを一目で示す (control_flow レイヤー)
- **補助陣ラベル**: 補助陣の上端に弧テキスト (if/else/for/while/loop/match パターン
  /クロージャ |params|)。Rust が kind から組み立てて IR に載せる (二度書きを避ける —
  既存の match パターン・クロージャ params が role に詰まっている経路の延長)。
  signatureFit と同じ近似で省略 (charWidthRatio 0.65、min 6px、default 9px)、
  type_info レイヤー (シグネチャと同階層)
- レビュー Critical 対応: `applyFilter` (SSR) が entrySigns / ringLabels もレイヤー
  フィルタすべき (`.magia` で hide できる規約)
- 凡例にも追記済み — 意匠を変えても凡例が自動追従する規約 (前半と同じ)

  本質は配置の問題なので 4.9 (空きスペースレイアウト) で吸収する
