# Phase 4.5 — ワークスペース俯瞰 (ファイル横断・最大ズームアウト)

## 出典

- オーナー方針 (2026-06-11): 「全体のシグネチャで見たり」 — ピン中心ビューの「最大ズームアウト」相当
- 親計画: Phase 4.0〜4.4 で「同ファイル内」までは完成。本計画はその境界を越える

> 本計画は **イメージ感の段階**。詳細は 4.1〜4.4 の実装で見えてきた地形をもとに精緻化する。

## 目的

dev-server に「ワークスペース俯瞰モード」を加える。
ファイル単位の魔法陣群を「地形」のように並べ、ファイル境界・モジュール階層を視覚化する。
ピンが立っているファイルは強調、その他は遠景としてフェードさせる。

## イメージ感 (オーナー指定の全体像)

- 最大ズームアウト = ファイル全体・ワークスペース全体を「シグネチャ盾の地図」として
- ファイル境界 = 地形のフチ。同モジュールのファイルは近接、別モジュールは離れて
- ピンが立っているファイルは中心、その他は距離フェード (Phase 4.1 の発想をファイル単位に拡張)
- クリックで「ファイル内俯瞰 (Phase 4.1)」にズームイン

## スコープ (薄め)

### やる方向

- ファイル横断スキャン (Phase 4.0 の FunctionIndex をワークスペース化)
- ファイル単位のシグネチャ盾 (関数群の縮小魔法陣をタイル状に配置)
- モジュール階層を距離成分に追加 (Phase 4.2 の `LocalProximity` を `WorkspaceProximity` に拡張)
- `?scope=workspace|file|focus` URL パラメータでスケール切替 (ピン打ち直しではなく「視野半径切替」)

### やらない方向

- 大規模ワークスペース (>1000 関数) の最適化 (Phase 4.6 以降でストリーミング化を検討)
- 外部クレートまで含めた俯瞰 (本ワークスペース内に閉じる)

## 設計上の発想

- Phase 4.1 が「関数のピン」だったのに対し、4.5 では「ファイル/モジュールのピン」も導入
- ピンの粒度切替が「ズーム軸」の正体になる
- レイアウトは依然として決定論 (名前ソート + 階層ソート)

## 受け入れ基準 (粗)

- [ ] ワークスペース内の全ファイルがシグネチャ盾の地図として表示される
- [ ] ファイルクリックでファイル内俯瞰 (Phase 4.1) にズームイン
- [ ] モジュール階層が距離に反映される

## 後続候補

- 巨大ワークスペース対応 (ストリーミング SVG、LOD)

3. `WorkspaceProximity` 拡張

5. テスト + Stage 1/2 + 完了処理
