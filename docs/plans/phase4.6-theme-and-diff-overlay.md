# Phase 4.6 — テーマ拡張と Spell Diff 重ね合わせ (Phase 4 系列の仕上げ)

## 出典

- オーナー方針 (2026-06-11): 「機能を厚くしていこう」 — Phase 4 系列の仕上げ的なバスケット
- 親計画: Phase 4.0〜4.5 が完成した上での味付け

> 本計画は **イメージ感の段階**。詳細は 4.1〜4.5 の実装で見えてきたUI上の課題をもとに精緻化する。

## 目的

Phase 4 系列でできあがった「ピン中心 + ワークスペース俯瞰 + 呼び出しジャンプ」の土台に対し、視覚的・体験的な厚みを加える:

- syntect テーマ切替 (ソースビューの見た目)
- 魔法陣カラーパレット切替 (色覚アクセシビリティ含む)
- Phase 3.2 Spell Diff の重ね合わせ (フォーカス関数の before/after を同じピン位置で切替表示)
- ベルカ式 (Phase 3.5) との同時表示・スタイル軸切替

## イメージ感

- 表示の見た目はユーザ好みで調整可能に (ダーク/ライト、色覚特性)
- 同じピン位置で時間軸 (Spell Diff) や様式軸 (ミッドチルダ/ベルカ) を切替えられる
- Phase 4 で作ったピン中心モデルが「時間軸 / 様式軸 / 視野半径軸」の交差点になる

## スコープ (薄め)

### やる方向

- syntect テーマ切替 UI + URL `?theme=light|dark|<custom>`
- 魔法陣パレット切替 (`?palette=` 既定 / 色覚補助 / 高コントラスト)
- フォーカスに対する Spell Diff overlay (`?diff=HEAD~1` のようなリビジョン指定)
- ピン中心 × ベルカ式の同時切替 (`?style=midchilda|belka|both`)
- アクセシビリティ強化: ARIA、キーボード操作、書き起こし統合

### やらない方向

- 完全な配色エディタ (固定セットでよい)
- diff の3者比較 (左右2者 + overlay まで)

## 設計上の発想

- 各「軸」(時間・様式・視野半径) を直交させる
- 軸の組み合わせを URL に反映 (`?pin=foo&theme=dark&diff=main&style=midchilda&scope=focus`)
- 軸が増えても UI が破綻しないよう、パレット (右上 UI) をタブ化

## 後続候補

- 記号語彙の拡張 (notes §10)
- 外部公開用のパーマリンク生成
