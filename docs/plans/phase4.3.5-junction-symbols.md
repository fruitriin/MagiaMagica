# Phase 4.3.5 — 結節点シンボル (分岐・ループ記号の移設)

> **状態: 未着手 (イメージ感のみ — 実装着手時に精緻化する)**
> **前提: Phase 4.3 (Vue SSR 一本化) の完了後** (オーナー指定 2026-06-12)

## オーナー要望 (2026-06-12、Phase 4.1 サイクル中)

> 分岐シンボルが補助リングの中央にあるけど、これメイン側リング上、補助リングに
> 伸びる結節点に小さい円＋シンボルのほうが関数呼び出しなのか分岐なのか
> わかりやすそう

## イメージ

- 現状: 分岐 Y 字・ループ三角は**補助リングの中央**に描かれる (spec §6.1.3)。
  補助リングと召喚印 (外部呼び出しの円) が遠目に似ており、図を読むときに
  「これは分岐の腕か、関数呼び出しか」の判別に視線移動が要る
- 案: メインリング上の**結節点** (接続線がメインリング (anchor operation 位置)
  から補助リングへ伸びる起点) に**小さい円 + 記号**のバッジを置く。
  召喚印 (効果カテゴリ色の塗り円) とは「白地に黒記号の小バッジ」で対比される
- 補助リング中央は空く — 4.0.6 後半の補助陣ラベル (関数名スコープの表示) と
  併せて中央の使い途を再設計できる

## 実装時に精緻化する論点

- 記号の種類: 分岐 Y 字 / ループ三角をバッジサイズ (半径 6〜8px 想定) に縮めた
  意匠。anchor operation のドットと同位置になるため、ドットとバッジの統合
  (制御 Operation のドット自体をバッジ化する) も検討
- if 連鎖・match の複数の腕が同じ anchor を共有する場合、バッジは1個
  (結節点は腕ごとではなく制御構造ごと)
- レイアウト衝突 (Phase 1.8 の衝突回避) との整合 — バッジは操作ドットより
  大きいため、リング上の他要素との间隔を再評価
- spec §6.1.3 の意匠規約の更新 ([break] — Rust SVG レンダラは 4.3 で削除済みの
  前提なので、Vue (SSR 共用) コンポーネントだけを変更する)
- 凡例パネル (4.0.6) のシンボルサンプルが自動追従することを確認
