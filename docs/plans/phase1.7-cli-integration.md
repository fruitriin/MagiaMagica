# Phase 1.7 — CLI 統合と磨き (M7)

## 出典

- `project-docs/magia/spec-v0.1.md` §10.1 (Phase 1 スコープ)
- `project-docs/magia/tech-selection-v0.1.md` §2.5 (clap), §3 M7

## 目的

`magia-cli` クレートに、これまでのマイルストーン (M1〜M6) を統合する CLI を実装する。Phase 1 配布物としての完成度に磨きをかけ、README / fixtures / エラーメッセージを整える。

## スコープ

### やること

- `clap` 4.x の derive ベースでサブコマンド実装:
  - `magia render <FILE> --fn <NAME> [--layers control_flow,effects,type_info] [-o OUT.svg]`
    - `--layers` 省略時は3レイヤー全てを描画
    - `-o` 省略時は標準出力に SVG を吐く
  - `magia list <FILE>` — ファイル内の関数一覧を出力
  - `magia emit-ir <FILE> --fn <NAME>` — IR を JSON で標準出力
- エラーハンドリングは `anyhow` で集約。ユーザー向けに親切な日本語メッセージ:
  - 関数未発見 → 「ファイル内の関数一覧: ...」と候補を提示
  - 構文エラー → 行番号と文脈を表示
- `--version` / `--help` (clap が自動生成)
- ファイル拡張子チェック (`.rs` 以外は警告)
- `fixtures/` を拡充して以下を置く:
  - **合成サンプル** 8〜10 個: 純粋関数 / IO / network / db / filesystem / unsafe / async / Result / match / ループ
  - **自己ホスティングサンプル**: `magia-core` / `magia-rust` / `magia-cli` 自身の代表関数を `magia render` で描画するサンプルコマンドを README に1つ載せる (オーナー確定: 「このリポジトリそのものをスナップショットとして使う」方針)
- リポジトリルートに `README.md` を**日本語**で作成 (オーナー未確定だがコミットログ規約と整合):
  - プロジェクト概要 (1パラグラフ、`project-docs/concept.md` から引用)
  - インストール手順 (`cargo install --path crates/magia-cli`)
  - 使用例 (各サブコマンドのコマンドラインと、`fixtures/simple_compute.rs` の出力 SVG のリンク)
  - **自己ホスティング例**: `magia render crates/magia-core/src/layout.rs --fn layout` で MagiaMagica 自身の関数を魔法陣化するコマンド
  - Phase 1 のスコープと制限事項
  - ライセンス表示 (MIT)
  - `project-docs/magia/spec-v0.1.md` への参照
- (英語 README が必要になったら Phase 2 以降で `README.en.md` として追加)
- 統合テスト (`magia-cli/tests/`):
  - 全 fixture に対して `render` が exit code 0 で SVG を出すこと
  - `list` が関数名を全て列挙すること
  - `--fn` で存在しない関数を指定したらエラーメッセージに候補が含まれること

### やらないこと

- dev-server / watch モード (Phase 2)
- 複数関数を1枚に描画 (Phase 1 スコープ外)
- フィルター DSL (Phase 2)
- PR コメント投稿 / CI 統合 (Phase 3)
- ライセンスファイル整備 (tech-selection §4 で未定)

## 設計上の判断

- `--emit-ir` は `magia emit-ir` サブコマンドとして独立させる (デバッグ用)。`render --emit-ir` フラグは混乱を招くので採用しない
- ロガーは Phase 1 では入れない (必要になれば `env_logger` を Phase 2 で導入)
- `--layers` は カンマ区切り文字列を `Vec<String>` で受け、未知のレイヤー名はエラーにする (タイポ防止)
- 出力 SVG のヘッダコメントに `<!-- generated by magia-cli vX.Y.Z -->` を入れる (insta スナップショットは差分が出るので fixtures は CLI ではなく `magia-core` レベルで取る)

- [ ] `magia --help` / `--version` が動作する
- [ ] 3サブコマンド (`render` / `list` / `emit-ir`) が全 fixture で動く
- [ ] 関数未発見時に候補一覧が表示される
- [ ] `cargo test --workspace` が全通過 (統合テスト含む)

- [ ] `README.md` が存在し、最初の使用例が再現可能
- [ ] `fixtures/` が 8 個以上の合成 Rust サンプルを持つ
- [ ] README に自己ホスティング (`magia` 自身のコード描画) の例が掲載されている
- [ ] README が日本語で書かれ、MIT ライセンス表示を含む
